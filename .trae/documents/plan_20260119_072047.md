1. 创建 `src/infrastructure/teo/teo.service.ts` 文件，封装 TeoClient 实例化和相关操作
2. 创建 `src/infrastructure/teo/teo.module.ts` 文件，将 TeoService 注册为一个 NestJS 模块
3. 将 TeoModule 导入到 AppModule 中，使 TeoService 可以被整个应用使用
4. 修改 `AccessRuleController`，通过依赖注入的方式使用 TeoService
5. 移除控制器中的硬编码 TeoClient 实例

这样设计的好处：

* 集中管理 TeoClient 配置和实例

* 支持依赖注入，便于测试和维护

* 符合 NestJS 最佳实践

* 实现了关注点分离，控制器只负责处理 HTTP 请求和响应

最终的 TeoService 将提供：

* 初始化和配置 TeoClient

* 封装 TEO SDK 的各种方法

* 提供统一的错误处理

* 支持配置管理
